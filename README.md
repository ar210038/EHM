# Environmental Health Monitor (EHM)

## Overview

Environmental Health Monitor (EHM) is an IoT and Android-based system that measures real-time environmental conditions and converts them into a meaningful environmental health risk assessment.

The system combines sensor data, cloud connectivity, machine learning, and a rule-based recommendation engine to help users understand their immediate environmental exposure and take appropriate actions.

Unlike conventional weather or AQI applications that provide city-level information, EHM focuses on localized environmental conditions in indoor spaces such as homes, classrooms, offices, and workplaces.

The system also includes an intelligent suggestion engine that provides actionable recommendations based on detected environmental risks.

## System Architecture

The system is divided into five layers:

### 1. Sensor Layer

Collects environmental data using:

* DHT22 (Temperature and Humidity)
* MQ135 (Air Quality Estimation)
* Noise Sensor
* OLED Display (Local Display)

### 2. Edge Layer

ESP32 Microcontroller

Responsibilities:

* Reads sensor values
* Performs basic preprocessing
* Displays readings locally
* Sends data to cloud through WiFi

### 3. Cloud Layer

Firebase Realtime Database

Responsibilities:

* Stores environmental data
* Synchronizes data in real time
* Supports historical data retrieval
* Enables cloud-based notifications

### 4. Analytics Layer

#### Health Risk Engine

* Normalizes sensor values
* Calculates Environmental Health Risk Score
* Identifies contributing risk factors

#### Machine Learning Model

* Random Forest Classifier
* Predicts environmental risk category

#### Suggestion Engine

* Analyzes risk contributors
* Generates actionable recommendations
* Prioritizes multiple environmental issues

### 5. Application Layer

Android Application (Kotlin)

Features:

* Live Dashboard
* Risk Score Visualization
* Historical Reports
* Push Notifications
* Environmental Recommendations

## Data Flow

1. Sensors collect environmental data.
2. ESP32 processes and transmits data via WiFi.
3. Firebase stores and synchronizes the data.
4. Android application retrieves real-time readings.
5. Health Risk Engine calculates the Health Risk Score.
6. Random Forest model predicts the environmental risk category.
7. Suggestion Engine generates actionable recommendations.
8. Results are displayed through dashboards, reports, alerts, and suggestions.

## Health Risk Scoring

The Health Risk Score is generated by combining normalized environmental parameters using weighted contributions based on their potential impact on environmental exposure and human comfort.

### Normalization

All sensor values are converted to a normalized scale between 0 and 1.

Parameters:

* AQI
* Noise Level
* Temperature
* Humidity

### Weighted Formula

Health Risk Index (HRI)

* AQI → 35%
* Noise → 30%
* Temperature → 20%
* Humidity → 15%

Final Score:

Health Risk Score = HRI × 100

### Risk Categories

| Score Range | Category  |
| ----------- | --------- |
| 0–30        | Safe      |
| 31–60       | Moderate  |
| 61–80       | Unhealthy |
| 81–100      | Dangerous |

## Machine Learning Model

A Random Forest classifier is used to classify environmental conditions into risk categories.

Since a unified real-world dataset containing AQI, noise, temperature, humidity, and environmental risk labels does not exist, the training dataset is constructed by combining publicly available environmental datasets and generating labels using WHO and EPA environmental safety guidelines.

### Input Features

* AQI
* Noise Level
* Temperature
* Humidity

### Output

* Safe
* Moderate
* Unhealthy
* Dangerous

### Model Development

* Dataset preparation using public environmental datasets
* Label generation using WHO and EPA thresholds
* Model training using Scikit-learn
* ONNX conversion for mobile deployment

### Deployment

* Trained using Python
* Converted to ONNX format
* Executed on Android using ONNX Runtime

## Intelligent Suggestion System

The system includes a rule-based recommendation engine that provides real-time actionable suggestions based on environmental conditions.

Unlike the risk score, which summarizes overall environmental exposure, the suggestion engine explains why the risk is elevated and what actions can be taken to improve conditions.

### Example Suggestions

#### High AQI

* Close windows
* Reduce outdoor exposure
* Use air purification if available

#### High Noise

* Move to a quieter location
* Reduce nearby noise sources
* Use hearing protection when necessary

#### High Temperature

* Improve ventilation
* Use cooling systems
* Stay hydrated

#### High Humidity

* Increase airflow
* Improve room ventilation
* Reduce prolonged exposure to enclosed spaces

### Multiple Risk Factors

When multiple environmental factors contribute significantly to the risk score, the system prioritizes and displays multiple recommendations.
